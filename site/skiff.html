../skiff

static site generator, purely written in Rust.

(2021)

a static site generator, written in Rust. traditionally, JavaScript is inevitable in modern web development. with technological acceleration, everything changes rapidly. we implicitly forced to catch up with the latest technology and discard outdated one.


to inherite Longtermism philosophy, solutioning for technological resilience and sustainability. by omitting JavaScript and only using plaintext database called indental and tablatal to generate HTML/CSS. almost any device is able to intepret and display them hence the website generated by skiff is vary accessible. this (might) encourage to rethink about technological waste and responsibility for an environment.

Usage.

add data content in /rustlib/database/lexicon.ndtl.

add an cover image file in /media/images/*.(jpg|png) then add image data in/rustlib/database/journals.ndtl.

(currently, database path is not configurable) then cargo run at rustlib folder.

skiff will manage to parse and outputs HTML file where it'll be located at /site

for additional details, create file name identical to entity in lexicon.ndtl eg.skiff.htm

the output file will be injected skiff.htm after parsing lexicon.ndtl.

lexicon.ndtl

  (indentation is matter)
SKIFF HOST : home TYPE : index BREF : static site generator, purely written in Rust. YEAR : 2021 TAGS Development STACK Rust BODY a static site generator, written in {https://www.rust-lang.org/ Rust}. LINK source : https://github.com/karnpapon/skiff
Keyword Type Indentation Description Notes
SKIFF entity 0 the output filename will be generated by this keyword space is not allowed. use hyphen instead.
HOST host name 2 name of parent default is home. -
TYPE type 2 type of the keyword normally should be "index" -
BREF short description 2 short description. -
YEAR year 2 define year here. -
TAGS tags [2, 4] tags for related keyword, tag name should be on the newline with 4 indentations. -
STACK tech stack [2, 4] stack name should be on the newline with 4 indentations. this will be displayed on right column at details level page. -
BODY description [2, 4] body content details should be on the newline with 4 indentations. this will be displayed after an cover image at details level page. -
SOURCE source url [2, 4] url content details should be on the newline with 4 indentations. this will be displayed on right column at details level page. -

journals.ndtl

19L12 - skiff 12021 skiff_generator
Keyword Notes
skiff keyword to link to lexicon.ndtl entities.
12021 image name at /media/images/*.(jpg|png)
skiff_generator image description use in alt image's attribute and will be displayed beneath cover image at details level page.
